# OurCV 产品文档

> 文档版本：v1.0
> 对应产品版本：OurCV v4
> 整理日期：2026年7月6日
> 产品形态：隐私优先、无需登录、可离线编辑的静态简历工具

## 一、产品概述

### 1. 产品定位

OurCV 是一个打开网页即可编辑的静态简历工具。它将简历的排版规则、模块结构和导出能力预先产品化，用户不需要掌握 HTML、设计软件或复杂的在线编辑器，也不需要注册账号、上传个人信息。

产品的核心价值可以概括为：

- **开箱即用**：进入页面后直接在简历纸张上编辑，不经过注册、建档和多步配置。
- **隐私优先**：简历正文和照片默认保存在当前浏览器，不上传业务服务器。
- **降低排版门槛**：用模板和智能分页代替复杂的字号、间距、页边距配置。
- **免费可迁移**：支持本地备份、导入和清除，避免被账号或会员体系锁定。
- **易于分发**：由原生 HTML、CSS 和 JavaScript 构成，解压后即可使用，也可部署为静态网站。

### 2. 产品背景

简历制作是高频刚需，但现有方案通常存在三类问题：

1. 商业简历工具往往以“免费编辑、付费导出”为核心模式，用户完成内容后才发现模板或导出能力需要付费，决策成本和挫败感较高。
2. GitHub 上的开源简历项目通常需要安装依赖、修改代码或自行部署，对普通求职者门槛较高，也更适合开发者或高度定制化场景。
3. Word、Photoshop 等通用工具自由度高，但缺少简历场景的版式约束，用户需要自行处理字体、间距、分页和信息层级。

OurCV 最初源于一次真实求职需求：先用 HTML 完成一份简历，再将其中可复用的设计规则和编辑流程抽象为产品，使不懂代码的用户也能直接使用。

### 3. 目标用户

- 希望快速制作中文或英文简历的求职者；
- 重视个人信息安全、不愿将简历上传至第三方平台的用户；
- 不熟悉设计软件、前端代码或复杂排版工具的用户；
- 希望免费使用、自由备份并长期保留简历数据的用户；
- 需要在单栏与双栏版式之间快速切换的用户。

### 4. 核心使用场景

1. 用户首次打开页面，直接在示例简历上替换内容。
2. 用户根据岗位特点，在经典单栏和经典双栏之间切换。
3. 用户调整主题、模块顺序和内容密度，形成一页或多页 A4 简历。
4. 用户进入保存态检查最终效果，并导出 PDF 或 PNG。
5. 用户下载 JSON 备份，在其他设备或浏览器中继续编辑。
6. 用户完成使用后，一键清除当前浏览器中的简历和照片。

## 二、产品原则

### 1. 先编辑，后配置

用户打开产品后直接进入编辑页面，不设置欢迎页、登录页和多步建档流程。纸张本身就是编辑器，减少“表单填写—切换预览—返回修改”的往返操作。

### 2. 用智能能力代替复杂参数

产品不向普通用户暴露大量字号、行距、页边距等专业参数，而是通过“智能一页”在可读范围内自动调整排版密度，并根据内容量扩展为多页。

### 3. 本地优先

文字、照片、偏好设置和模板状态均保存在浏览器本地。产品不依赖账号体系和业务后端完成核心编辑流程。

### 4. 内容与版式分离

经典单栏和经典双栏共用同一份数据。用户切换模板时只改变呈现方式，不需要重复填写，也不会丢失内容。

### 5. 克制而非堆叠

产品只保留简历制作的核心能力。复杂 AI 功能、商业服务和高级定制在验证价值前不进入主流程，避免增加一次性用户的学习成本。

## 三、迭代时间线

### 2026年7月4日 00:00—12:00｜从个人模板到产品 MVP

项目首先完成静态简历编辑器的基础设计和实现方案，将原有 HTML 简历从“修改代码”转化为“页面直接编辑”。

本阶段重点解决四个问题：

- 建立工作经历、项目经历、教育经历等结构化数据模型；
- 使用 `contenteditable` 实现纸面直接编辑；
- 使用 `localStorage` 自动保存文字和偏好；
- 建立编辑态与保存态，保证编辑控件不进入最终简历。

同时完成头像本地读取、经历增删排序、恢复示例、PDF 打印和 PNG 导出的基础能力，并使用通用示例替换原始私人简历内容。

### 2026年7月4日 12:00—14:00｜明确产品定位与商业边界

在 MVP 可用后，产品从“个人求职工具”转向“可供他人使用的简历产品”，形成以下判断：

- 产品应保持免费、无需登录和低门槛；
- 进入网站后应直接开始制作简历，而不是先经过多级页面；
- 隐私安全应成为核心心智，而不仅是一项技术实现；
- 产品名称确定为 OurCV，强调“属于每个人的简历工具”；
- 网站服务与 AI 简历评审可作为后续能力，但不影响基础编辑器免费使用。

这一阶段同步形成域名部署、静态托管、用户反馈和增值服务的初步设想。

### 2026年7月4日 14:00—19:25｜视觉升级与双模板架构

通过分析英文简历产品的字体、留白和信息层级，产品增加第二套真实可编辑模板，并将模板能力从“换皮”升级为“同一数据、不同信息架构”。

形成 `OurCV-v3.zip` 阶段版本，主要能力包括：

- **经典单栏**：适合多数中文求职场景，按工作、项目、教育顺序展示。
- **经典双栏**：使用编辑型版式，主栏呈现工作与教育，侧栏呈现项目与能力工具。
- 两套模板共用统一状态模型，切换时不丢失数据。
- 增加暖砂红、白灰、白蓝和白绿四套主题。
- 引入本地 Noto Sans SC 和 Newsreader 字体，改善中英文排版。
- 双栏支持左右栏宽度拖动调整。
- 新增个人定位语、能力工具和可编辑页脚。
- 建立不同模板下的密度方案与导出适配。

### 2026年7月4日 19:25—21:13｜v4 信息降噪与编辑体验优化

在双栏版本基础上继续收敛视觉元素、增强模块控制，形成当前 `OurCV-v4.zip`。

本轮完成的需求包括：

1. 经典单栏支持工作经历与项目经历调整上下顺序。
2. 双栏“能力工具”支持移除和恢复；移除只隐藏栏目，不销毁已填写数据。
3. 删除“个人简介 / 00”等装饰性数字，降低视觉噪声。
4. 将“代表项目”统一改为“项目经历”。
5. 删除 `PROJECT / 01` 形式，仅在项目名称前保留 `01、02……` 序号。
6. 将品牌口号“你的经历，你来落笔”移至 OurCV 右侧，并使用更醒目的斜体衬线字体。
7. 放大个人简介正文，缩小标题与正文之间的无效间距。
8. 删除工作经历、项目经历、教育经历等模块标题后的章节序号。
9. 放大岗位、职责方向和工作地点等次级信息，提高阅读层级。
10. 编辑态允许通过占位信息识别电话、邮箱和所在地，保存及导出态只呈现用户填写内容，不展示 `PHONE、EMAIL、BASED` 等英文标签。
11. 经典双栏补充标准一寸照比例的照片上传区域，与单栏共用本地照片数据。

本轮同时延续了项目经历栏目的移除与恢复机制，并补充双栏能力工具的同类交互，使“隐藏模块”和“删除内容”成为两种不同操作。

- 压缩包中未发现个人简历、联系方式、证件照、私钥或其他个人材料；

## 四、当前产品能力

### 1. 简历编辑

- 直接点击纸张文字进行编辑；
- 支持姓名、目标方向、联系方式、个人主页、个人简介和个人定位语；
- 支持工作经历、项目经历和教育经历的新增、删除与条目排序；
- 支持经历成果描述的逐条新增和删除；
- 空字段在保存及导出状态自动折叠；
- 支持恢复隐私安全的示例数据。

### 2. 模板与视觉

- 经典单栏和经典双栏两套可编辑模板；
- 两套模板共用同一份内容；
- 暖砂红、白灰、白蓝和白绿四套主题；
- 本地中英文字体；
- 双栏侧栏宽度可拖动；
- 双栏项目自动编号；
- 标准一寸照比例的头像区域。

### 3. 模块管理

- 经典单栏支持栏目顺序调整；
- 项目经历支持隐藏和恢复；
- 双栏能力工具支持隐藏和恢复；
- 隐藏栏目不会删除内容，恢复后数据仍然存在；
- 保存与导出状态不显示空栏目和编辑控件。

### 4. 智能版面

- 自动尝试五档排版密度；
- 优先保持一页 A4；
- 内容不适合继续压缩时，自动扩展至 2—10 页；
- 超过十页时，根据内容量提示大致需要删减的字数；
- 内容变化后提示用户重新执行智能排版。

### 5. 本地数据

- 通过 `localStorage` 自动保存；
- 450 毫秒防抖，减少高频写入；
- 头像通过 `FileReader` 转为 Data URL 后保存在本地；
- 支持下载 JSON 备份；
- 支持导入 OurCV 备份；
- 支持一键清除本机简历与照片；
- 对旧版数据提供默认字段补全。

### 6. 导出

- 支持导出 PNG；
- 支持按 A4 自动分页导出 PDF；
- 导出前自动切换至无编辑控件的保存状态；
- 导出组件加载失败时，PDF 使用浏览器打印功能兜底；
- 文件名默认使用用户姓名。

### 7. 隐私说明

- 页面内提供独立隐私抽屉；
- 解释本地存储、备份、清除和基础访问日志；
- 不接入广告脚本；
- 核心编辑流程无需注册和后端接口。

## 五、产品架构

### 1. 技术结构

OurCV 使用无框架静态架构：

```text
index.html        应用外壳、工具栏、模板选择和隐私说明
styles.css        编辑台、单双栏模板、主题、密度和打印样式
app.js            数据模型、渲染、交互、本地存储和导出逻辑
assets/           品牌图标及本地字体
模板简历.html      兼容入口
```

### 2. 数据模型

产品使用一份 JSON 状态描述简历：

```text
profile           个人信息、简介、照片、页脚
work[]            工作经历
projects[]        项目经历
education[]       教育经历
skills[]          能力工具
preferences       模板、主题、密度、页数、隐藏栏目和栏目顺序
```

经典单栏与经典双栏分别拥有渲染函数，但读取同一状态。所有用户输入在进入 HTML 前进行转义，减少导入内容破坏页面结构的风险。

### 3. 核心数据流

```text
用户直接编辑
  ↓
更新内存状态
  ↓
防抖写入 localStorage
  ↓
模板渲染 / 智能分页
  ↓
保存态预览
  ↓
PDF、PNG 或 JSON 本地文件
```

照片不经过上传接口，由浏览器读取后与简历状态一起保存在本机。

### 4. 外部依赖

核心编辑、保存、模板和打印能力不依赖外部服务。本地打包了：

- Noto Sans SC；
- Newsreader。

PNG 和自动 PDF 导出使用随项目分发的固定版本组件：

- `html2canvas 1.4.1`；
- `jsPDF 2.5.1`。

组件在首次导出时从本地目录加载；若组件不可用，PDF 可回退到浏览器打印。

## 八、衡量指标

当前版本尚未接入数据统计。后续若在不破坏隐私原则的前提下进行产品验证，建议优先关注：

- 首次进入至完成第一次有效编辑的转化率；
- 保存态预览率；
- PDF/PNG 导出成功率；
- 智能分页使用率和成功率；
- 模板切换率；
- 本地备份下载率；
- 用户完成一份简历所需时长；
- 用户反馈中的排版、导出和隐私问题占比。

不建议收集简历正文、联系方式、头像或可识别具体求职者的行为数据。

## 九、产品结论

OurCV 已从一次个人简历制作需求，发展为具备统一数据模型、双模板渲染、本地保存、模块控制、智能分页和多格式导出的可用产品。它最有辨识度的不是功能数量，而是三个选择：

1. 不要求用户注册和上传简历；
2. 不把复杂排版参数转嫁给用户；
3. 不用付费墙限制基础编辑和导出。

v4 已具备公开展示和小范围试用条件。许可证、仓库隔离说明、README 更新和导出依赖本地化已经完成；完成关键流程测试后，可以作为正式开源预览版发布。
